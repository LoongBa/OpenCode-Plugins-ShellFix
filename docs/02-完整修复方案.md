# 完整修复方案：ShellFixPlugin

## 一、问题定义

### 1.1 背景

Windows PowerShell 下 OpenCode 存在两个顽固问题：

| # | 问题 | 根因 | 表象 |
|---|------|------|------|
| 1 | Linux `export` 语法不兼容 | LLM 训练数据以 bash 为主，Agent 频繁输出 `export KEY=VAL`，PowerShell 无法执行 | 报错：`'export' 不是内部或外部命令` |
| 2 | 非交互子进程不加载 `$PROFILE` | OpenCode 的 bash 工具在非交互模式下运行，不执行用户的 PowerShell Profile 文件 | 中文乱码、自定义环境变量/别名失效 |

### 1.2 需求

1. **自动语法转换**：执行 shell 命令前拦截，将 `export` 语句批量转为 PowerShell `$env:KEY="VAL"` 写法。
2. **全局前置环境注入**：每条命令自动追加 UTF-8 编码配置，预填充 CI 静默环境变量。
3. **约束**：无第三方依赖、仅处理 shell 工具、零人工干预、全局一次配置永久生效。

## 二、设计方案

### 2.1 技术栈

| 层次 | 选型 |
|------|------|
| 载体 | OpenCode 原生 TypeScript 插件 |
| 运行时 | Bun（OpenCode 内置，自动编译执行 .ts 文件） |
| 钩子 | `shell.env` + `tool.execute.before` |
| 类型定义 | `@opencode-ai/plugin`（仅开发时类型检查，选装） |

### 2.2 架构

```
┌─────────────────────────────────────────────────────────────┐
│                    ShellFixPlugin                            │
│                                                             │
│  ┌──────────────────────┐  ┌──────────────────────────────┐ │
│  │  shell.env           │  │  tool.execute.before         │ │
│  │  (进程初始化)         │  │  (命令执行前)                 │ │
│  │                      │  │                              │ │
│  │  仅注入环境变量       │  │  1. 拼接编码前置脚本          │ │
│  │  output.env.KEY=VAL  │  │  2. 检测 export → 转换为     │ │
│  │                      │  │     $env:KEY="VAL"           │ │
│  │  14 个 CI 静默变量    │  │  3. 非 export → 仅加编码前缀  │ │
│  └──────────────────────┘  └──────────────────────────────┘ │
│                                                             │
│  两个钩子独立解耦，各司其职                                   │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 钩子详解

#### 钩子 A：shell.env

| 项目 | 内容 |
|------|------|
| 触发时机 | 每次 shell 进程初始化（AI 工具 + 用户终端） |
| 官方签名 | `(input: { cwd: string }, output: { env: Record<string, string> }) => Promise<void>` |
| 能力边界 | **只能**设环境变量，**不能**拼命令 |
| 本插件用途 | 预注入 14 个 CI 全局环境变量 |

预注入的环境变量：

| 变量 | 值 | 用途 |
|------|----|------|
| CI | `"true"` | 标记 CI 环境，抑制交互式提示 |
| DEBIAN_FRONTEND | `"noninteractive"` | apt 等工具非交互模式 |
| GIT_TERMINAL_PROMPT | `"0"` | 禁止 Git 弹出凭据提示 |
| GCM_INTERACTIVE | `"never"` | Git 凭据管理器非交互 |
| HOMEBREW_NO_AUTO_UPDATE | `"1"` | Homebrew 不自动更新 |
| GIT_EDITOR / EDITOR | `":"` | 设置编辑器为 no-op |
| VISUAL | `""` | 覆盖 $VISUAL |
| GIT_SEQUENCE_EDITOR | `":"` | rebase 编辑为 no-op |
| GIT_MERGE_AUTOEDIT | `"no"` | 合并不打开编辑器 |
| GIT_PAGER / PAGER | `"cat"` | Git 分页输出为 cat（不阻塞） |
| npm_config_yes | `"true"` | npx 自动 yes |
| PIP_NO_INPUT | `"1"` | pip 非输入模式 |
| YARN_ENABLE_IMMUTABLE_INSTALLS | `"false"` | Yarn 允许可变 install |

#### 钩子 B：tool.execute.before

| 项目 | 内容 |
|------|------|
| 触发时机 | 每次工具调用前 |
| 官方签名 | `(input: { tool: string }, output: { args: any }) => Promise<void>` |
| 能力边界 | **可以**修改 `output.args` 改写命令参数 |
| 本插件用途 | ① 注入编码前缀 ② 转换 export 语法 |

**职责 1：编码前缀注入**

```powershell
# 在每条命令最前面自动拼接：
$OutputEncoding=[Text.UTF8Encoding]::new($false);
[Console]::OutputEncoding=[Text.UTF8Encoding]::new($false);
```

- `$OutputEncoding`：控制管道输出编码（OpenCode 捕获 stdout 走这条路径）
- `[Console]::OutputEncoding`：控制台显示编码
- 两条都必须设，少一条中文都可能乱码

**职责 2：export → $env 转换**

转换规则：

| Bash 原文 | PowerShell 转换结果 |
|-----------|-------------------|
| `export FOO=bar` | `$env:FOO="bar";` |
| `export FOO="hello world"` | `$env:FOO="hello world";` |
| `export FOO='hello $PATH'` | `$env:FOO="hello `$PATH";` |
| `export KEY1=val1 KEY2=val2` | `$env:KEY1="val1";$env:KEY2="val2";` |
| `export FOO=bar && make` | `$env:FOO="bar"; && make` |
| `export FOO=bar; echo $FOO` | `$env:FOO="bar"; echo $FOO` |

### 2.4 数据流

```
                   Agent 生成命令
                        │
                        ▼
           ┌────────────────────────┐
           │  tool.execute.before   │
           │                        │
           │  以 export 开头？       │
           │   ├── 是 → 正则提取 KV │
           │   │       → 转 $env:   │
           │   │       → 拼接编码   │
           │   └── 否 → 直接拼接编码 │
           └────────────────────────┘
                        │
                        ▼
           ┌────────────────────────┐
           │    PowerShell 执行     │
           │   $OutputEncoding=UTF8 │
           │   中文不乱码           │
           │   $env:KEY="VAL" 生效   │
           └────────────────────────┘
```

### 2.5 执行流程

```
OpenCode 启动
    │
    ├─ 加载 ShellFixPlugin（.opencode/plugins/shell-fix.ts）
    │
    ├─ Agent 输出 "export CI=true"
    │
    ├─ shell.env → 环境变量已预注入（选做，非必须）
    │
    ├─ tool.execute.before 拦截命令
    │   ├─ 检测到 export
    │   ├─ 提取 CI=true
    │   ├─ 转为 $env:CI="true";
    │   ├─ 拼接编码前缀
    │   └─ 覆盖 output.args.command
    │
    └─ PowerShell 执行修复后命令 → 无报错、不乱码
```

## 三、完整源代码

详见 `../src/shell-fix.ts`。代码要点：

### 3.1 变量覆盖说明

| 转换步骤 | 关键函数 | 说明 |
|----------|----------|------|
| 外层引号剥离 | `val.slice(1, -1)` | 移除 KEY="VAL" 或 KEY='VAL' 的引号 |
| 转义还原 | `val.replace(/\\(["\\])/g, "$1")` | `\"` → `"`, `\\` → `\` |
| PS 安全转义 | `escapePSDoubleQuoted()` | `` ` `` → ``` `` ```, `$` → `` `$ ``, `"` → `""` |
| 命令拼接 | `ENCODING_PREFIX + assigns + suffix` | 编码前缀 + 环境赋值 + 原始剩余命令 |

### 3.2 正则设计

**外层匹配（判断是否 export 行）：**
```
/^export\s+((?:\w+=(?:"(?:[^"\\]|\\.)*"|'(?:[^'\\]|\\.)*'|\S+)\s*)+)(.*)$/s
```

**内层解析（提取各 KEY=VAL 对）：**
```
/\w+=(?:"(?:[^"\\]|\\.)*"|'(?:[^'\\]|\\.)*'|\S+)/g
```

### 3.3 安全策略

| 措施 | 说明 |
|------|------|
| 仅处理 bash/pwsh 工具 | `if (tool !== "bash" && tool !== "pwsh") return;` |
| 仅匹配行首 export | 避免错误转换命令中间或注释中的 export |
| 格式异常时保底 | export 行无法解析时仅注入编码，保留原命令 |
| PowerShell 转义 | 防 `$` 变量注入、`"` 字符串截断、`` ` `` 逃逸 |

## 四、安装与配置

### 4.1 插件放置

**方式一：全局级（推荐，所有项目生效）**

```bash
# 创建全局插件目录
mkdir -p ~/.config/opencode/plugins/

# 复制插件
cp shell-fix.ts ~/.config/opencode/plugins/
```

OpenCode 会自动发现该目录下的所有 `.ts` / `.js` 文件并加载。

**方式二：项目级（仅当前项目生效）**

```bash
# 创建项目插件目录
mkdir -p .opencode/plugins/

# 复制插件
cp shell-fix.ts .opencode/plugins/
```

**方式三：手动注册（如果自动发现未生效）**

在 `opencode.json` 或 `opencode.jsonc` 的 `plugin` 数组中添加路径：

```json
{
  "plugin": [
    "oh-my-openagent@latest",
    ".opencode/plugins/shell-fix.ts"
  ]
}
```

### 4.2 验证安装

重启 OpenCode 后，检查是否生效：

1. Agent 输出 `export FOO=bar` 后不再报错
2. 中文输出不乱码
3. 环境变量 `$env:CI` 已存在（`echo $env:CI` 返回 `true`）

### 4.3 卸载

删除对应位置的 `shell-fix.ts` 文件，重启 OpenCode。

## 五、局限性

| 场景 | 是否支持 | 说明 |
|------|----------|------|
| `export KEY=plain_value` | ✅ | 基础用法 |
| `export KEY="quoted value"` | ✅ | 双引号值 |
| `export KEY='single quoted'` | ✅ | 单引号值 |
| `export K1=v1 K2=v2` | ✅ | 多变量一起 export |
| `export KEY=val && command` | ✅ | 跟 && 链接 |
| `export KEY=val \` + 换行 + `CONT` | ❌ | 多行续行不支持 |
| `export -n KEY` (unexport) | ❌ | 极少见，跳过 |
| `export` 无参数（打印所有） | ❌ | -p 等价，跳过 |
| `export KEY=$OTHER_REF` | ⚠️ | 转为 `$env:KEY="`$OTHER_REF"` 字面量而非引用值 |
| `export` 在命令中间 | ❌ | 只匹配行首 |

## 六、与原方案的核心差异

| 维度 | 原方案 | 本修复方案 |
|------|--------|-----------|
| shell.env output | `envCtx.preCommand` + `envCtx.vars` | `output.env.KEY = "VALUE"` |
| preCommand 字段 | 虚构 | 不存在，移至 tool.execute.before |
| 编码注入 | shell.env 中幻想 | tool.execute.before 中真实拼接 |
| $PROFILE 加载 | 每条命令加载 | 废弃，替换为精确编码命令 |
| 正则表达式 | 不支持引号值 | 完整支持引号、转义 |
| PS 安全转义 | 无 | `$` → `` `$ ``, `"` → `""`, `` ` `` → ``` `` ``` |
| 边界处理 | 无 | 异常格式保底、命令类型过滤、防逃逸 |
