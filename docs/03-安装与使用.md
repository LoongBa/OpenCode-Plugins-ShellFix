# 安装与使用指南

当前版本：**v1.1.0**

## 环境要求

| 依赖 | 版本要求 | 说明 |
|------|----------|------|
| OpenCode | ≥ 1.0（支持插件系统） | 内置 Bun 运行时，自动编译 .ts 文件 |
| 操作系统 | Windows（PowerShell 5.1+） | 解决的核心问题是 PowerShell 兼容性 |
| @opencode-ai/plugin | ≥ 1.17.13（选装） | 仅用于 TypeScript 类型检查，非运行必需 |

---

## 安装步骤

### 步骤 1：放置插件文件

将 `src/shell-fix.ts` 复制到 OpenCode 的插件目录。

**全局安装（推荐）**——所有项目生效：

```bash
# 确认全局插件目录存在
mkdir -p ~/.config/opencode/plugins/

# 复制插件
cp shell-fix.ts ~/.config/opencode/plugins/
```

**项目级安装**——仅当前项目生效：

```bash
# 在项目根目录创建 .opencode/plugins/
mkdir -p .opencode/plugins/

# 复制插件
cp shell-fix.ts .opencode/plugins/
```

### 步骤 2：重启 OpenCode

关闭当前 OpenCode 会话并重新启动。

### 步骤 3：验证安装

#### 验证 1：插件版本环境变量

在 OpenCode 中执行：

```powershell
echo $env:SHELLFIX_VERSION
```

预期输出：`1.1.0`

如果返回空或 undefined，说明插件未加载。

#### 验证 2：环境变量预注入

在 OpenCode 中执行：

```powershell
echo $env:CI
```

预期输出：`true`

如果返回空或 undefined，说明插件未加载。

#### 验证 3：export 自动转换

在 OpenCode 中让 Agent 执行：

```bash
export MY_TEST_VAR="HelloWorld"
echo $env:MY_TEST_VAR
```

预期输出：`HelloWorld`

测试没有 `export` 命令报错即成功。

#### 验证 4：中文编码

```powershell
Write-Output "你好，世界！测试中文不乱码"
```

预期输出：`你好，世界！测试中文不乱码`

#### 验证 5：多项 export

让 Agent 执行：

```bash
export DB_HOST=localhost DB_PORT=5432 DB_NAME=mydb
echo "$env:DB_HOST:$env:DB_PORT/$env:DB_NAME"
```

预期输出：`localhost:5432/mydb`

#### 验证 6：状态指令 `/shellfix`

让 Agent 执行 `/shellfix` 或 `#shellfix`，预期输出：

```
[ShellFix] v1.1.0
  ├ Status: Active
  ├ Export syntax: $env:KEY=VAL auto-convert
  ├ Encoding: UTF-8 forced prefix
  └ CI vars: 18 pre-injected
```


## FAQ

### Q：如何确认插件已加载？

三种方式：

1. **环境变量** — 执行 `echo $env:SHELLFIX_VERSION`，输出 `1.1.0`
2. **状态指令** — 让 Agent 执行 `/shellfix`，显示状态面板
3. **启动日志** — 检查 OpenCode 日志 `~/.local/share/opencode/log/opencode.log`，搜索 `[ShellFix]`

### Q：插件是否会影响非 Windows 系统？

不会。`shell.env` 钩子注入的 CI 环境变量是跨平台通用的（除 `DEBIAN_FRONTEND` 外，它在 Linux 上只是不生效，不会报错）。`tool.execute.before` 的编码前缀使用 PowerShell 语法，在 Linux/macOS 的 bash 工具中会自动报错：

```bash
$OutputEncoding=...  # bash 中报错：command not found
```

**因此建议仅在 Windows 环境中启用此插件。** 或者可以增强插件使其在非 Windows 平台静默退出：

```typescript
"tool.execute.before": async (input, output) => {
  if (process.platform !== "win32") return; // 非 Windows 跳过
  // ...
}
```

### Q：和现有插件冲突吗？

插件使用 `shell.env` 和 `tool.execute.before` 两个标准钩子。如果有其他插件也使用相同钩子，OpenCode 按加载顺序依次执行所有插件的同名钩子。多个插件可以通过 `output` 对象链式传递修改结果。

如果出现冲突，调整插件加载顺序或在 `opencode.json` 中明确指定顺序。

### Q：export 语法转换是否 100% 兼容 bash 语义？

不完全。见"局限性"章节。主要差异：

| Bash 语义 | 插件行为 | 差异 |
|-----------|----------|------|
| `export FOO=$BAR`（引用变量） | `$env:FOO="`$BAR"`（字面量） | `` `$BAR `` 在 PS 中是字面量而非变量引用 |
| `export FOO=$(cmd)`（命令替换） | 不支持 | 正则无法匹配 `$()` 和反引号 |
| 多行 export | 不支持 | `export FOO=bar \` + 换行 + `BAZ=qux` |

这些场景在 LLM 生成的命令中极少出现，实际影响有限。

### Q：$PROFILE 不加载了，自定义环境变量怎么办？

通过 `shell.env` 钩子手动注入。如果需要自定义环境变量，在 `shell-fix.ts` 的 `CI_ENV_VARS` 对象中添加即可：

```typescript
const CI_ENV_VARS: Record<string, string> = {
  // ...默认变量...
  MY_CUSTOM_KEY: "my_custom_value",
  ANOTHER_VAR: "another_value",
};
```

## 卸载

删除插件文件后重启 OpenCode 即可：

```bash
# 全局安装的
rm ~/.config/opencode/plugins/shell-fix.ts

# 项目安装的
rm .opencode/plugins/shell-fix.ts
```

## 开发与调试

### 本地修改

插件是纯 TypeScript 文件，修改后保存即可生效（重启 OpenCode 或重新加载插件）。

### 测试方法

使用 `shell-env-test.ts` 测试脚本验证 `shell.env` 环境注入：

```typescript
// test/shell-env-test.ts
const plugin = await import("../src/shell-fix");
const instance = await plugin.ShellFixPlugin({} as any);
const output = { env: {} } as any;
await instance["shell.env"]!({ cwd: "/test" } as any, output);
console.log("Env vars:", Object.keys(output.env).length);
console.assert(output.env.CI === "true", "CI should be true");
```

### 日志调试

在插件关键路径添加 `console.log`：

```typescript
"tool.execute.before": async (input, output) => {
  console.log(`[ShellFix] intercepting ${input.tool} command`);
  // ...
  console.log(`[ShellFix] converted: ${output.args.command}`);
}
```

日志会输出到 OpenCode 的日志目录（`~/.local/share/opencode/log/`）。
