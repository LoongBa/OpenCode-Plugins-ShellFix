## 一、交付内容

### 1.1 /kickme 通知系统 ✅

**功能：** 用户注册关键词/正则匹配规则，当用户消息或 LLM 回复中匹配时弹出 toast 通知 + 可选提示音。

**实现文件：** `src/lib/state.ts`、`src/shell-fix-tui.ts`、`src/shell-fix.ts`

**核心 API：**

| 函数 | 位置 | 说明 |
|------|------|------|
| `KickmeRule` 类型 | state.ts:73 | id / label / matchType / pattern / title / message / sound / scope |
| `getKickmeRules()` | state.ts:321 | 读取所有规则 |
| `addKickmeRule()` | state.ts:326 | 添加规则，返回 id |
| `removeKickmeRule()` | state.ts:335 | 删除规则 |
| `toggleKickmeRule()` | state.ts:345 | 开关规则 |
| `setKickmeSound()` | state.ts:355 | 开关提示音 |
| `checkKickmeRules(text, scope)` | tui.ts:1085 | 规则匹配 + toast/notify |

**命令语法：**
```
/kickme                           → 列出所有规则
/kickme add <keyword> <title> <message>
/kickme rm <id>                   → 删除规则
/kickme on|off <id>               → 开关规则
/kickme sound <id> on|off         → 开关提示音
```

**事件集成：**
- `session.next.prompted` → `checkKickmeRules(text, "user")` — 用户消息触发
- `session.next.text.ended` → `checkKickmeRules(text, "llm")` — LLM 回复触发

### 1.2 标签 Timeline 浏览 ✅

**功能：** 按时间倒序查看笔记，支持标签过滤和时间范围过滤。

**实现文件：** `src/lib/template-store.ts`、`src/shell-fix-tui.ts`、`src/shell-fix.ts`

**核心 API：**

| 函数 | 位置 | 说明 |
|------|------|------|
| `listNotesByTime(limit?, since?)` | template-store.ts:319 | 所有笔记按时间倒序 |
| `queryNotesByTime(prefix, limit?, since?)` | template-store.ts:331 | 按标签 + 时间过滤 |

**命令语法：**
```
/note timeline                    → 显示最近 20 条
/note timeline --limit 50         → 最多 50 条
/note timeline --since 2025-01-01 → 从指定日期开始
/note timeline #架构#              → 按标签过滤
/note timeline #架构# --since 3d   → 组合过滤
```

### 1.3 会话级自动注入 ✅

**功能：** `auto.mode === "prompt"` 时，TUI 启动弹窗让用户选择本次会话启用的自动化模块。

**实现文件：** `src/shell-fix-tui.ts`

**实现细节：**
- TUI 初始化后 800ms 弹出 `DialogSelect` 对话框
- 用户点击切换模块（toggle），选完后点「✅ 完成选择」
- 模块状态通过 `setAutoModule()` 持久化到 `PluginState.auto.modules`
- 现有 `experimental.chat.system.transform` 钩子自动读取最新状态注入 system prompt

## 三、架构评审意见（Oracle）

### 3.1 /auto 管线扩展 → 推迟到 v1.9

**问题：**
- `replace_input`/`replace_output` 因插件 API 不支持 payload mutation，无法真正实现替换语义 — 设计为死路
- `notify` 与 kickme、`collect` 与 autoCollectTags 功能重复，增加维护负担
- 四类新 action 中仅 `inject_system` 有价值，而它已存在

**决策：** 推迟到 v1.9，重新设计

### 3.2 会话注入弹窗需要多选设计

**问题：** 原设计 `DialogSelect` 每次只能 toggle 一个模块，用户需多次操作

**方案：** 多轮 `DialogSelect` + 刷新弹窗 + 「✅ 完成选择」退出

### 3.3 Timeline 设计稳健

**评价：** 笔记已有 `created: ISO` 时间戳，基础设施完整。`listNotesByTime` / `queryNotesByTime` 附加值高，无架构风险。
